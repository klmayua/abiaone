## 1. CANONICAL STRUCTURE METHODOLOGY

Canonical pages are determined by grouping all 115 unique pages into logical page families:
- **Viewport variants:** `-mobile` suffix = same page, mobile layout → consolidate to 1 responsive page
- **Iteration variants:** `-2`, `-3` suffixes = designer iterations of same page → consolidate to 1 canonical page
- **Role variants:** `-admin` suffix = admin dashboard variant → keep separate if functionally distinct
- **No assumptions:** Each canonical page requires evidence of distinct user purpose
